A. 礦砂採集

這題是很典型的連續背包問題。結論非常簡單：只要不斷貪婪地將單位重量價值最大的礦砂放進背包，直到背包已滿或所有礦砂均已被放入，就能最大化總價值。

一個簡單的做法是將所有礦砂依照單位重量價值由大到小排序，依序放入背包直到滿即可。複雜度是O(nlog⁡n)O(n \log n)O(nlogn)。

線性做法

雖然並不是本題的考點，但有一個值得一提的 O(n)O(n)O(n) 做法存在。以下為了方便說明，假定所有的 xix_ixi​ 均相異。

  1. 隨機選擇一個礦砂 iii，並將所有的礦砂分為單位價值 ≥xi\ge x_i≥xi​ 與 <xi< x_i<xi​ 兩堆。
    分歧判斷：
    • 若單位價值 ≥xi\ge x_i≥xi​ 的礦砂可以塞滿背包，則將 <xi< x_i<xi​ 的礦砂全刪除，並往大的 xxx 找答案。
    • 反之則將價值 ≥xi\ge x_i≥xi​ 的礦砂全刪除，往小的 xxx 找答案。
      搜尋完後得到的值代表「總價值最大時，背包中單位重量價值最小的礦砂」，再用這個價值去反求答案。

演算法與 quickselect 非常相似，期望時間複雜度為 O(n)O(n)O(n)。

B. 村莊與小徑

這題是一道最短路徑的問題，但給定的限制中有幾個點要注意：

  1. 給定的圖是 DAG（有向無環圖）。
  2. 邊權可能是負的。

條件 1 保證這張圖存在拓樸排序，求解可用 top-down DP，或找出拓樸排序後用 bottom-up DP。

// ord = 拓樸排序的點順序
// fr[x] = 連到 x 的所有點

fill(dis, dis+n+1, INF);
dis[1] = 0;
for (int i=1; i<=n; i++) {
  int p = ord[i];
  // 找所有能連到 p 的點 j，更新 p 的最短距離
  for (Edge j: fr[p]) dis[p] = min(dis[p], dis[j.to] + j.weight);
}

以上的做法複雜度是 O(n+m)O(n+m)O(n+m)，可以在限制內解完所有測資。

錯誤／部分解法

值得一提的是這題有幾個預設的錯誤解法：

  1. Dijkstra: 不能處理負權的最短路。
  2. Bellman–Ford / SPFA: 這類演算法在最糟情況下的複雜度是 Θ(nm)\Theta(nm)Θ(nm)，只能過比較小的測資。（註：SPFA 在圖為隨機生成的情況下跑得非常快，要防止讓 SPFA 過基本上需要構造測資。不過總而言之確實存在 SPFA 無法解過的測資）

C. 樣本解析

把給定的 XXX 拆成兩個不相交集合 X1X_1X1​ 和 X2X_2X2​，再暴力求出所有樣本和 X1X_1X1​ 及 X2X_2X2​ 的交集狀況就可以了。由於集合的拆法有 O(2m)O(2^m)O(2m) 種且枚舉完以後需要花 O(n)O(n)O(n) 檢查，總時間複雜度為 O(2m×n)O(2^m \times n)O(2m×n) (mmm 為 XXX 的字元種類數)。

要怎麼實作得比較簡潔是這一題的重點，這裡給一些小技巧供參考：

若用 int 儲存集合，可以用一個 for 迴圈枚舉所有的子集合。

D. 水果包裝

觀察與結論

換個方式來思考這一題：假設每次機器在選定最小重量的袋子後，由你決定放入哪顆水果。那麼這題存在一個非常簡單的結論：如果給定的包裝結果是可能的，只要重量越輕的水果越先放，總是能達成該包裝結果。

或許這個結論不難猜，但要好好證明也許會花點時間。假設題目給定的包裝是存在解的，也就是存在一個解的序列：
(b1,w1),(b2,w2),⋯ ,(bn,wn).(b_1, w_1), (b_2, w_2), \cdots,(b_n, w_n).(b1​,w1​),(b2​,w2​),⋯,(bn​,wn​).
這裡用 (bi,wi)(b_i, w_i)(bi​,wi​) 代表第 iii 步驟的水果放在第 bib_ibi​ 袋，且放的水果重量為 wiw_iwi​。

我們將焦點放在這個解序列中，由「放入相同袋子的兩步驟」所形成的子區間：
(b,w1),⋯ ,(b,w2).(b, w_1), \cdots, (b, w_2).(b,w1​),⋯,(b,w2​).
可以發現上述序列若 w1>w2w_1 > w_2w1​>w2​，我們其實可以交換這兩個步驟的順序，且合法的解仍然存在：
(b,w2),⋯ ,(b,w1),⋯(b, w_2), \cdots, (b, w_1), \cdots(b,w2​),⋯,(b,w1​),⋯
儘管交換後 bbb 一度變輕導致 (b,w1)(b, w_1)(b,w1​) 可能會比原本 (b,w2)(b, w_2)(b,w2​) 的位置還前面，我們發現在放完 w1w_1w1​ 後，只要這個子區間的水果還沒被放完，bbb 就不會是最輕的，因此這樣的操作是合法的。（當然，對放水果順序的影響只會出現在注目的子區間內，子區間外的所有順序都無需改動。）

經過複數次上述「交換同袋子水果順序」的操作以後，我們可以將任意一個解的序列交換成對每個袋子而言收到的水果重量皆為從小到大。由於任意解的順序都可以經由以上的交換得到同一個序列，若從小拿到大仍然無法滿足題目給定的包裝，代表無解。

實作 O( n(log n + log m) )

實作上我們會需要一個資料結構可以

  1. 知道現在最輕且編號最小的袋子 bbb。
  2. 將還沒放入 bbb 且最輕的水果放進 bbb 裡，並更新 bbb 的重量。

只要將包裝結果中的每個袋子的水果依重量排序，便可隨時查出該將哪個水果放入袋子中；找出最輕的袋子與更新重量部分，可以用堆積或 std::priority_queue 維護，在 O(log⁡m)O(\log m)O(logm) 時間完成每個操作。因此整體的複雜度為 O(n(log⁡n+log⁡m))O(n(\log n+\log m))O(n(logn+logm))。

E. 共同朋友

O(n³)

首先這題有一個簡單的 naive O(n3)O(n^3)O(n3) 做法:

for (int i=1; i<=n; i++) {
  for (int j=i+1; j<=n; j++) {
    if (have_common_friends(i, j))
      answer++;
  }
}

計算 have_common_friends() 最簡單的方法是寫一個迴圈，需時 O(n)O(n)O(n)。

O(n³/B)

我們利用 std::bitset f[i] 的第 jjj 個 bit 記錄 iii 和 jjj 是不是朋友，以優化上述 have_common_friends() 的迴圈實作。想求 iii 和 jjj 有沒有共同朋友時，可用 std::bitset 的 and 運算與 method any() (i.e. (f[i] & f[j]).any()) 得出。

由於 std::bitset 的實作方式為 bit array，執行時間會比一般的陣列快數十倍以上。當然自己用陣列來實作 bit array 也是可行的：

unsigned bit(int x) {
  return 1u << x;
}

void add_friend(int i, int j) {
  j--;
  f[i][j/32] |= bit(j%32);
}

bool have_common_friends(int i, int j) {
  int len = (n+31) / 32;
  for (int k=0; k<len; k++) {
    if (f[i][k] & f[j][k]) return true;
  }
  return false;
}

掃描線

做完上述變換後，在不考慮題目問的是整數點的情況下，可以把題目簡化成這樣：

「平面上有一些與 x,yx,yx,y 軸平行的矩形，每塊矩形皆有權重。定義一個點的點負重為所有覆蓋該點的矩形的權重和，請求出點負重的最大值。」

這個問題是一個含有區間更新的 2D RMQ 問題，但因為所有矩形為已知，可以離線利用掃描線掃其中一軸，並用線段樹或平衡二元搜尋樹維護掃描線上的最大值，時間複雜度 O(nlog⁡n)O(n \log n)O(nlogn)。
